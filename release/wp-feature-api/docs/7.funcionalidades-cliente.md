# 7. Funcionalidades del Lado del Cliente

> **Guía para registrar y usar funcionalidades JavaScript que se ejecutan en el navegador del usuario.**

---

Mientras que las clases PHP principales gestionan principalmente funcionalidades del lado del servidor (ejecutadas dentro del backend de WordPress), la API de Funcionalidades también soporta funcionalidades del lado del cliente implementadas en JavaScript y ejecutadas en el navegador del usuario.

---

## El Paquete `@automattic/wp-feature-api`

Todas las interacciones del lado del cliente (registro, ejecución, descubrimiento) dependen del paquete `@automattic/wp-feature-api`. Este paquete proporciona las funciones necesarias y gestiona un registro del lado del cliente que refleja el del servidor.

### Configuración Requerida

El paquete `@automattic/wp-feature-api` está diseñado para trabajar como un singleton, proporcionado por el plugin principal "WordPress Feature API". Para usarlo en el JavaScript de tu propio plugin:

1. **Plugin Principal Activo:** Asegúrate de que el plugin "WordPress Feature API" esté instalado y activo en tu sitio.
2. **Dependencia del Script:** Cuando registres tu archivo JavaScript del plugin, debes declarar `'wp-features'` (el handle del script para el SDK del cliente proporcionado por el plugin principal) como una dependencia.
3. **Externals de Webpack:** Si estás usando webpack (o un bundler similar), debes configurar `@automattic/wp-feature-api` como un módulo external. Esto le dice a tu bundler que no incluya el código del SDK en tu bundle y que en su lugar use la versión proporcionada globalmente por el plugin principal (vía `window.wp.features`).

### Ejemplo de Configuración

**Encolando el script en WordPress:**

```php
<?php
function mi_plugin_encolar_scripts() {
    wp_enqueue_script(
        'mi-plugin-feature-script',
        plugins_url( 'build/index.js', __FILE__ ),
        array( 'wp-features' ), // Dependencia del Feature API
        '1.0.0',
        true
    );
}
add_action( 'wp_enqueue_scripts', 'mi_plugin_encolar_scripts' );
```

**Configuración de webpack:**

```javascript
// webpack.config.js
externals: {
    '@automattic/wp-feature-api': 'wp.features',
    // Otros externals de WordPress...
}
```

> 📚 Para instrucciones de configuración detalladas, incluyendo ejemplos de `wp_enqueue_script` y configuración de webpack, consulta el [README en el paquete client](../packages/client/README.md).

## Registro de Funcionalidades

### Cómo Registrar

Define todo el objeto de funcionalidad (incluyendo `id`, `name`, `description`, función `callback` y opcionalmente `input_schema`, `output_schema`, `categories`, etc.) directamente en tu código JavaScript. Luego, usa la función `registerFeature` de `@wp-feature-api/client` para añadirlo al registro del lado del cliente.

### Ejemplo de Registro

```javascript
import { registerFeature } from '@automattic/wp-feature-api';

const insertarParrafo = {
    id: 'core/insertar-parrafo',
    name: 'Insertar Bloque de Párrafo',
    description: 'Añade un nuevo bloque de párrafo al editor.',
    type: 'action',      // Tipo de acción para el cliente
    location: 'client',  // IMPORTANTE: Indica que se ejecuta en el navegador
    callback: () => {
        // ... lógica para insertar bloque de párrafo ...
        console.log('¡Párrafo insertado!');
        return { success: true };
    },
    // Esquemas opcionales
    input_schema: {
        type: 'object',
        properties: {
            contenido: {
                type: 'string',
                description: 'El contenido del párrafo a insertar.'
            }
        }
    },
    output_schema: {
        type: 'object',
        properties: {
            success: { type: 'boolean' }
        }
    },
    categories: ['editor', 'bloques'],
    // Elegibilidad opcional
    is_eligible: () => {
        // Verificar si el editor está disponible
        return typeof wp !== 'undefined' && wp.data && wp.data.select('core/editor');
    }
};

registerFeature(insertarParrafo);
```

## Ejecución de Funcionalidades

### Cliente a Cliente

El código del lado del cliente puede descubrir y ejecutar directamente otras funcionalidades registradas del lado del cliente usando las funciones `getRegisteredFeatures` y `executeFeature` de `@automattic/wp-feature-api`.

```javascript
import { getRegisteredFeatures, executeFeature } from '@automattic/wp-feature-api';

// Obtener todas las funcionalidades registradas del lado del cliente
const funcionalidadesCliente = getRegisteredFeatures().filter(
    f => f.location === 'client'
);

// Ejecutar una funcionalidad del lado del cliente
async function ejecutarFuncionalidadCliente() {
    try {
        const resultado = await executeFeature('core/insertar-parrafo', {
            contenido: 'Hola desde el cliente!'
        });
        console.log('Resultado:', resultado);
    } catch (error) {
        console.error('Error:', error);
    }
}
```

### Ejecución Activada por el Servidor

Una funcionalidad puramente del lado del servidor (callback PHP) *no puede* ejecutar directamente una funcionalidad del lado del cliente (callback JavaScript). Sin embargo, un proceso del lado del servidor *puede orquestar* la ejecución de una funcionalidad del lado del cliente.

#### Mecanismo:

1. El proceso del lado del servidor (por ejemplo, un agente de IA interactuando vía REST API, o un handler de solicitud PHP) descubre la funcionalidad del lado del cliente. Esto puede requerir pasar las funcionalidades permitidas del lado del servidor al proceso del lado del servidor.

2. El servidor envía una respuesta o mensaje al entorno del cliente indicándole que ejecute la funcionalidad. Esto podría ser a través de un campo de respuesta API dedicado (como `client_action` en la demostración) o a través de un mensaje WebSocket.

3. El código del lado del cliente recibe esta instrucción y usa la función `executeFeature` de `@automattic/wp-feature-api` para ejecutar el callback JavaScript correspondiente con los argumentos proporcionados.

#### Ejemplo del Patrón

```javascript
// En el cliente - recibiendo instrucción del servidor
async function manejarAccionServidor(respuestaServidor) {
    if (respuestaServidor.client_action) {
        const { action_id, params } = respuestaServidor.client_action;

        if (action_id === 'demo/log-message') {
            // Ejecutar la funcionalidad del lado del cliente
            const resultado = await executeFeature('demo/log-message', params);
            return resultado;
        }
    }
}
```

> 💡 **Nota:** La demostración del agente (`wp-feature-api-demo`) muestra este patrón. El backend del agente podría decidir llamar a la funcionalidad `demo/log-message`. Le dice al frontend del chat que luego usa `executeFeature('demo/log-message', { message: '¡Hola desde el agente!' })` desde la biblioteca del cliente para ejecutar la función JS. Consulta `demo/wp-feature-api-agent/src/agent/wp-feature-tool-provider.ts` para ver un ejemplo de cómo se usa `executeFeature` de la biblioteca del cliente en este contexto.

## Tabla de Comparación: Servidor vs Cliente

| Aspecto | Funcionalidades del Servidor | Funcionalidades del Cliente |
|---------|------------------------------|-----------------------------|
| **Lenguaje** | PHP | JavaScript |
| **Ubicación** | Backend de WordPress | Navegador del usuario |
| **Ejecución** | `$feature->call()` en PHP | `executeFeature()` en JS |
| **Acceso a Datos** | Base de datos, archivos | DOM, estado del navegador |
| **Acceso a WP** | API completa de WordPress | API de WordPress.js |
| **Persistencia** | Entre solicitudes | Durante la sesión actual |
| **Location** | `server` | `client` |

## Casos de Uso Comunes

| Caso de Uso | Descripción |
|-------------|-------------|
| **Interacciones del Editor** | Insertar bloques, guardar publicaciones, publicar contenido. |
| **UI Dinámica** | Mostrar modales, notificaciones, menús personalizados. |
| **Comunicación en Tiempo Real** | Actualizaciones en vivo sin recargar la página. |
| **Integración con APIs del Navegador** | Geolocalización, notificaciones del navegador, etc. |
