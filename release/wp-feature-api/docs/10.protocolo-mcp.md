# 10. Protocolo de Contexto de Modelo (MCP) - Adaptador

> **Guía sobre cómo la API de Funcionalidades se integra con el Model Context Protocol mediante el plugin wordpress-mcp.**

---

## 1. API de Funcionalidades - La Fuente de Verdad Local

La **API de Funcionalidades** es la forma uniforme en que WordPress core, plugins y temas registran *capacidades* que pueden invocarse programáticamente. Cada definición de `WP_Feature` incluye:

| Propiedad | Descripción |
|-----------|-------------|
| **id** | Identificador único de la funcionalidad. |
| **name** | Nombre legible para mostrar. |
| **description** | Descripción detallada de la funcionalidad. |
| **input_schema** | Definición JSON Schema de los parámetros de entrada. |
| **output_schema** | Definición JSON Schema del formato de salida. |
| **permission_callback** | Función para verificar permisos. |
| **callable** | La implementación ejecutable (`$feature->call()`). |

Dentro de WordPress, la API de Funcionalidades (vía `wp_get_features()` y funciones relacionadas) permanece como *el único* registro autoritativo. Nada se expone externamente hasta que un adaptador lo publica.

---

## 2. Plugin Adaptador MCP (`wordpress-mcp`)

El plugin [wordpress-mcp](https://github.com/Automattic/wordpress-mcp) es un Plugin Adaptador que mapea los objetos `WP_Feature` registrados al [Model Context Protocol (MCP)](https://modelcontextprotocol.io/introduction).

Una vez instalado y activado, convierte automáticamente tu sitio en un servidor MCP que hosts de LLM (Claude Desktop, extensiones de VS Code, etc.) pueden descubrir e invocar.

### Cómo Funciona el Adaptador

| Paso | Descripción |
|------|-------------|
| **1. Descubrimiento** | Lee el `WP_Feature_Registry` para obtener todas las funcionalidades registradas que han optado por exposición externa (vía un flag como `expose_via => [ 'mcp' ]`). |
| **2. Mapeo** | Convierte cada `WP_Feature` en la definición de *herramienta* o *recurso* MCP correspondiente: |
| | - `WP_Feature::$id` → MCP `id` |
| | - `WP_Feature::$name` → MCP `name` |
| | - `WP_Feature::$description` → MCP `description` |
| | - `input_schema` / `output_schema` → JSON Schemas MCP idénticos |
| **3. Ejecución** | Cuando un cliente emite una solicitud MCP: |
| | 1. Localiza la funcionalidad correspondiente. |
| | 2. Verifica permisos. |
| | 3. Invoca `$feature->call( $input )`. |
| | 4. Devuelve la respuesta en el formato que MCP espera. |

---

## 3. ¿Por Qué una Capa Adaptadora?

| Beneficio | Descripción |
|-----------|-------------|
| **Desacoplamiento** | Las capacidades centrales permanecen específicas de WordPress; las preocupaciones de protocolo viven en plugins separados y reemplazables. |
| **Experimentación** | Los nuevos estándares emergen rápidamente. Al aislar la lógica del protocolo, WordPress puede enviar adaptadores adicionales sin reescribir funcionalidades existentes. |
| **Exposición Selectiva** | Las funcionalidades individuales pueden declarar qué protocolos externos (si alguno) deben verlas. |

---

## 4. Más Allá de MCP - Adaptadores Futuros

MCP es actualmente el estándar principal para integraciones de IA/LLM, pero **no** es la única opción en el horizonte. Dado que los adaptadores son solo plugins, la comunidad puede implementar fácilmente soporte para:

| Protocolo | Descripción |
|-----------|-------------|
| **Endpoints OpenAPI estilo FastAPI** | Útil para integraciones servidor a servidor ligeras. |
| **Protocolo Agente a Agente (A2A) de Google** | Una especificación emergente para comunicación entre agentes. |
| **Cualquier otro protocolo** que gane tracción. |

Cada adaptador adicional seguiría el mismo patrón: leer del registro de la API de Funcionalidades, traducir metadatos a la especificación objetivo, aplicar permisos y ejecutar el callback de la funcionalidad.

---

## 5. Ejemplo de Exposición Opt-in

Esto aún no está implementado, pero el plan es añadir un flag a la función `register_feature()`:

```php
<?php
register_feature(
    array(
        'id'                => 'miplugin/crear_post',
        'name'              => 'Crear Publicación',
        'description'       => 'Crea una nueva publicación desde entrada estructurada.',
        'input_schema'      => $mi_input_schema,
        'output_schema'     => $mi_output_schema,
        'permission_callback' => 'puede_crear_publicaciones',
        'callable'          => 'miplugin_crear_post',
        // Exponer esta funcionalidad vía MCP *y* FastAPI una vez que esos adaptadores estén instalados.
        'expose_via'        => array( 'mcp', 'fastapi' ),
    )
);
```

---

## 6. Resumen

| Concepto | Descripción |
|----------|-------------|
| **API de Funcionalidades** | El registro canónico de capacidades invocables dentro de un sitio WordPress. |
| **Plugin wordpress-mcp** | Publica esas capacidades sobre el **Model Context Protocol**, haciéndolas consumibles por herramientas modernas de LLM. |
| **Patrón Adaptador** | Registra una vez con la API de Funcionalidades, y deja que los plugins adaptadores manejen el resto. |

> **TL;DR:** Registra una vez con la API de Funcionalidades, y deja que los plugins adaptadores manejen la exposición a diferentes protocolos.
