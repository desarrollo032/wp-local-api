# 4. Usar Funcionalidades

> **Guía para descubrir, consultar y ejecutar funcionalidades registradas (`wp_find_feature`, `wp_get_features`).**

---

Una vez que las funcionalidades están registradas, pueden ser recuperadas y ejecutadas usando funciones globales o la instancia del registro.

---

## Encontrar una Funcionalidad Específica (`wp_find_feature`)

Para obtener un solo objeto de funcionalidad por su ID completo (incluyendo el prefijo de tipo):

```php
<?php
/**
 * Encuentra una funcionalidad por su ID.
 *
 * @param string $feature_id El ID completo de la funcionalidad (ej: 'recurso-miplugin/titulo-sitio').
 * @return WP_Feature|null El objeto de funcionalidad si se encuentra y es elegible, null de lo contrario.
 */
function wp_find_feature( string $feature_id ): ?WP_Feature
```

### Ejemplo de Uso

```php
<?php
$funcionalidad_titulo = wp_find_feature( 'recurso-miplugin/titulo-sitio' );

if ( $funcionalidad_titulo ) {
    // La funcionalidad existe y es elegible
    $titulo = $funcionalidad_titulo->call();
} else {
    // Funcionalidad no encontrada o no elegible en el contexto actual
}
```

> ⚠️ **Nota:** `wp_find_feature` internamente verifica `is_eligible()`. Si la funcionalidad existe pero su callback `is_eligible` devuelve `false`, `wp_find_feature` devolverá `null`.

---

## Consultar Múltiples Funcionalidades (`wp_get_features`)

Para obtener un array de funcionalidades que coinciden con criterios específicos:

```php
<?php
/**
 * Obtiene funcionalidades basándose en una consulta.
 *
 * @param WP_Feature_Query|array|null $query Los argumentos de consulta o objeto WP_Feature_Query. Si es null, devuelve todas las funcionalidades elegibles.
 * @return WP_Feature[] Un array de objetos WP_Feature que coinciden y son elegibles.
 */
function wp_get_features( $query = null ): array
```

### Ejemplos de Uso

```php
<?php
// Ejemplo: Obtener todas las funcionalidades 'herramienta' elegibles en la categoría 'woocommerce'
$herramientas_woo = wp_get_features( array(
    'type'       => array( WP_Feature::TYPE_TOOL ),
    'categories' => array( 'woocommerce' ),
) );

foreach ( $herramientas_woo as $herramienta ) {
    echo 'Nombre de Herramienta: ' . esc_html( $herramienta->get_name() ) . "\n";
}

// Ejemplo: Obtener todas las funcionalidades elegibles
$todas_funcionalidades = wp_get_features();

// Ejemplo: Buscar funcionalidades relacionadas con "email de usuario"
$funcionalidades_email = wp_get_features( array(
    'search' => 'email de usuario',
) );
```

> ⚠️ **Nota:** `wp_get_features` también filtra resultados basándose en `$feature->is_eligible()`.

---

## La Clase `WP_Feature_Query`

Aunque puedes pasar un array a `wp_get_features`, también puedes instanciar `WP_Feature_Query` directamente para escenarios más complejos o autocompletado de tipo.

```php
<?php
$argumentos_consulta = array(
    'type' => array( 'recurso' ),
    'categories' => array( 'core', 'post' )
);
$consulta = new WP_Feature_Query( $argumentos_consulta );

$recursos_post_core = wp_get_features( $consulta );
```

### Argumentos de Consulta

| Argumento | Tipo | Descripción |
|-----------|------|-------------|
| `type` | `string[]` | Array de tipos (`recurso`, `herramienta`) a incluir. |
| `categories` | `string[]` | Array de slugs de categorías. Las funcionalidades deben pertenecer a *todas* las categorías listadas. |
| `location` | `string[]` | Array de ubicaciones (`server`, `client`) a incluir. |
| `input_schema` | `array` | Filtrar por campos del esquema de entrada. Requiere `fields` y opcionalmente `match`. |
| `output_schema` | `array` | Filtrar por campos del esquema de salida (misma estructura que `input_schema`). |
| `search` | `string` | Término de búsqueda para buscar en ID, nombre y descripción de funcionalidad (insensible a mayúsculas). |

### Filtrando por Campos de Esquema

```php
<?php
// Filtrar funcionalidades que tienen un campo 'post_id' en su esquema de entrada
$funcionalidades_post = wp_get_features( array(
    'input_schema' => array(
        'fields' => array( 'post_id' ),
        'match' => 'any', // 'any' (cualquiera) o 'all' (todas)
    ),
) );
```

## Ejecutar una Funcionalidad (`$feature->call()`)

Una vez que tienes un objeto `WP_Feature` (usualmente de `wp_find_feature`), lo ejecutas usando el método `call()`.

```php
<?php
/**
 * Llama a la funcionalidad.
 *
 * @param array $context Contexto/argumentos opcionales para pasar al callback de la funcionalidad.
 *                       Debe coincidir con el 'input_schema' de la funcionalidad.
 * @return mixed|WP_Error El resultado del callback de la funcionalidad, o un WP_Error en fallo
 *                        (ej: permiso denegado, error de validación, error de callback).
 */
// public function call( $context = array() )
```

### Ejemplo 1: Recurso sin entrada

```php
<?php
$feature = wp_find_feature( 'recurso-miplugin/tema-activo' );
if ( $feature ) {
    $nombre_tema = $feature->call(); // $context es opcional, por defecto []
    if ( is_wp_error( $nombre_tema ) ) {
        // Manejar error
    } else {
        // Usar $nombre_tema
    }
}
```

### Ejemplo 2: Herramienta con contexto de entrada

```php
<?php
$feature = wp_find_feature( 'herramienta-miplugin/actualizar-opcion' );
if ( $feature ) {
    $context = array(
        'nombre_opcion'  => 'blogname',
        'valor_opcion' => 'Mi Blog Increíble Actualizado',
    );
    $resultado = $feature->call( $context );
    if ( is_wp_error( $resultado ) ) {
        // Manejar error (ej: permisos, validación, actualización fallida)
    } elseif ( $resultado === true ) {
        // Opción actualizada exitosamente
    } else {
        // Actualización de opción fallida (devolvió false)
    }
}
```

### Ejemplo 3: Llamando una funcionalidad REST Alias

```php
<?php
$feature = wp_find_feature( 'recurso-post' ); // Alias para /wp/v2/posts/{id}
if ( $feature ) {
    $datos_post = $feature->call( array( 'id' => 1 ) ); // El contexto proporciona el {id}
    if ( is_wp_error( $datos_post ) ) {
        // Manejar error (ej: post no encontrado, permisos)
    } else {
        // Usar $datos_post (el array de post/stdClass de la API REST)
    }
}
```

## Resumen de Métodos

| Función/Método | Propósito |
|----------------|-----------|
| `wp_find_feature($id)` | Encuentra una funcionalidad específica por ID. |
| `wp_get_features($query)` | Obtiene múltiples funcionalidades que coinciden con criterios. |
| `$feature->call($context)` | Ejecuta la funcionalidad con el contexto dado. |
| `$feature->is_eligible()` | Verifica si la funcionalidad está disponible en el contexto actual. |
| `$feature->get_name()` | Obtiene el nombre legible de la funcionalidad. |
| `$feature->get_id()` | Obtiene el ID completo de la funcionalidad. |
