# 6. Categorías

> **Cómo agrupar y organizar funcionalidades relacionadas usando categorías.**

---

## Propósito y Uso

Las categorías proporcionan una forma de agrupar funcionalidades relacionadas, ayudando a la organización y descubrimiento.

| Beneficio | Descripción |
|-----------|-------------|
| **Organización** | Agrupa funcionalidades lógicamente (ej: `woocommerce`, `seo`, `gestion-usuarios`). |
| **Filtrado** | Permite que consumidores (como agentes de IA) soliciten funcionalidades solo de categorías relevantes, reduciendo el alcance de herramientas/recursos disponibles. |
| **Descubrimiento** | Proporciona contexto sobre el dominio al que pertenece una funcionalidad. |

---

## Definir Categorías

### Método 1: Implícitamente (Al Registrar Funcionalidades)

Cuando registras una funcionalidad, lista los slugs de categorías en el array `categories`. Si una categoría no existe, se creará automáticamente (usando el slug como nombre, en formato título).

```php
<?php
wp_register_feature( array(
    'id' => 'miplugin/mi-funcionalidad',
    // ... otros argumentos ...
    'categories' => array( 'mi-plugin', 'nueva-categoria', 'procesamiento-datos' ),
) );
```

### Método 2: Explícitamente (Recomendado para categorías compartidas/core)

Usa el filtro `wp_feature_default_categories` para definir categorías con nombres y descripciones apropiados desde la inicialización.

```php
<?php
add_filter( 'wp_feature_default_categories', function ( $categories ) {
    $categories['woocommerce'] = array(
        'name'        => __( 'WooCommerce', 'mi-plugin' ),
        'description' => __( 'Funcionalidades relacionadas con productos, pedidos, etc. de WooCommerce.', 'mi-plugin' ),
    );
    $categories['seo'] = array(
        'name'        => __( 'SEO', 'mi-plugin' ),
        'description' => __( 'Funcionalidades para gestionar Optimización para Motores de Búsqueda.', 'mi-plugin' ),
    );
    $categories['analytics'] = array(
        'name'        => __( 'Analíticas', 'mi-plugin' ),
        'description' => __( 'Funcionalidades relacionadas con estadísticas y analíticas del sitio.', 'mi-plugin' ),
    );
    return $categories;
} );
```

## Consultar por Categoría

Usa el parámetro `categories` en `wp_get_features()` o en la REST API:

### Consulta PHP

```php
<?php
// Obtener todas las funcionalidades en la categoría 'woocommerce'
$funcionalidades_woo = wp_get_features( array(
    'categories' => array( 'woocommerce' ),
) );

// Obtener funcionalidades que estén en AMBAS categorías
$funcionalidades_woo_post = wp_get_features( array(
    'categories' => array( 'woocommerce', 'post' ),
) );
```

### REST API

```bash
# Obtener todas las funcionalidades en la categoría 'woocommerce'
GET /wp-json/wp/v2/features?categories[]=woocommerce

# Obtener todas las herramientas en la categoría 'seo'
GET /wp-json/wp/v2/features?type[]=tool&categories[]=seo
```

## Estructura de Categoría

Cada categoría tiene la siguiente estructura:

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `slug` | string | Identificador único de la categoría (ej: `woocommerce`). |
| `name` | string | Nombre legible para mostrar (ej: `WooCommerce`). |
| `description` | string | Descripción de qué funcionalidades contiene la categoría. |
| `count` | integer | Número de funcionalidades en esta categoría. |

---

## Ejemplo de Respuesta de Categoría

```json
{
    "woocommerce": {
        "slug": "woocommerce",
        "name": "WooCommerce",
        "description": "Funcionalidades relacionadas con productos, pedidos y gestión de tienda.",
        "count": 15
    },
    "seo": {
        "slug": "seo",
        "name": "SEO",
        "description": "Funcionalidades para optimización de motores de búsqueda.",
        "count": 8
    }
}
```

## Mejores Prácticas

| Práctica | Descripción |
|----------|-------------|
| **Nombres Descriptivos** | Usa nombres claros que indiquen claramente el propósito de la categoría. |
| **Namespace como Categoría** | Considera incluir el namespace de tu plugin como una categoría (ej: `mi-plugin`). |
| **Categorías Genéricas** | Crea categorías genéricas para funcionalidades comunes (ej: `post`, `user`, `taxonomy`). |
| **Evitar Exceso** | No crees demasiadas categorías pequeñas; agrupa relacionadas cuando sea posible. |
