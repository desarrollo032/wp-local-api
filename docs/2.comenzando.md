# 2. Comenzando

> **Guía de instalación y primer ejemplo de uso de la API de Funcionalidades de WordPress.**

---

## Opciones de Instalación

### Opción 1: Usar Composer

Esta es la forma recomendada para desarrolladores de plugins que desean incluir la API de Funcionalidades de WordPress en sus plugins.

#### Paso 1: Agregar la dependencia

Añade la dependencia al `composer.json` de tu plugin:

```json
{
  "require": {
    "automattic/wp-feature-api": "^0.1.8"
  }
}
```

> ⚠️ **Nota:** Asegúrate de estar usando la última versión disponible. Revisa [Packagist](https://packagist.org/packages/automattic/wp-feature-api) para ver la versión más reciente.

#### Paso 2: Instalar dependencias

```bash
composer install
```

#### Paso 3: Incluir e inicializar en tu plugin

```php
<?php
// Incluye la API de Funcionalidades en el archivo principal de tu plugin
require_once __DIR__ . '/vendor/automattic/wp-feature-api/wp-feature-api.php';

// Registra tus funcionalidades cuando la API se inicialice
add_action('wp_feature_api_init', 'mi_plugin_registrar_funcionalidades');
```

---

## Ejemplo de Uso Básico

```php
<?php
// functions.php o tu archivo de plugin

// 1. Define una función callback para tu funcionalidad
function mi_plugin_obtener_titulo_sitio() {
    return get_bloginfo( 'name' );
}

// 2. Registra la funcionalidad cuando la API de Funcionalidades se inicializa
// Usar wp_feature_api_init asegura que la API esté completamente cargada
add_action( 'wp_feature_api_init', 'mi_plugin_registrar_funcionalidades' );

function mi_plugin_registrar_funcionalidades() {
    wp_register_feature( array(
        'id'          => 'miplugin/titulo-sitio', // ID único (namespace/nombre-funcionalidad)
        'name'        => __( 'Obtener Título del Sitio', 'mi-plugin' ),
        'description' => __( 'Recupera el título del sitio WordPress actual.', 'mi-plugin' ),
        'type'        => WP_Feature::TYPE_RESOURCE, // Recupera datos
        'callback'    => 'mi_plugin_obtener_titulo_sitio', // La función a llamar
        'permission_callback' => '__return_true', // Cualquiera puede acceder
        'categories'  => array( 'mi-plugin', 'info-sitio' ), // Categorías opcionales
    ) );
}

// 3. Después, puedes encontrar y llamar a la funcionalidad
add_action( 'admin_init', 'mi_plugin_usar_funcionalidad' );

function mi_plugin_usar_funcionalidad() {
    // Nota: el prefijo de tipo se añade automáticamente
    $funcionalidad_titulo = wp_find_feature( 'recurso-miplugin/titulo-sitio' );

    if ( $funcionalidad_titulo && $funcionalidad_titulo->is_eligible() ) {
        $titulo_sitio = $funcionalidad_titulo->call();
        // $titulo_sitio ahora contiene el resultado de get_bloginfo('name')
        // error_log( 'Título del Sitio: ' . $titulo_sitio );
    }
}
```

### Opción 2: Instalación Directa del Plugin

Si deseas contribuir a la API de Funcionalidades tú mismo:

1. Clona el [repositorio de la API de Funcionalidades de WordPress](https://github.com/Automattic/wordpress-feature-api):
   ```bash
   git clone https://github.com/Automattic/wp-feature-api.git
   ```

2. Coloca la carpeta del plugin (por ejemplo, `wp-feature-api`) en tu directorio `wp-content/plugins/`.

3. Activa el plugin "WordPress Feature API" a través del menú de Plugins de WordPress admin.

4. (Opcional) El plugin de demostración puede cargarse definiendo:
   ```php
   define( 'WP_FEATURE_API_LOAD_DEMO', true );
   ```
   en tu archivo `wp-config.php`.

## Verificación de la Instalación

Para verificar que la API de Funcionalidades está funcionando correctamente:

```php
<?php
add_action( 'admin_init', 'verificar_instalacion_feature_api' );

function verificar_instalacion_feature_api() {
    // Verificar que las funciones principales existen
    if ( function_exists( 'wp_register_feature' ) && function_exists( 'wp_get_features' ) ) {
        echo '<div class="notice notice-success"><p>✅ API de Funcionalidades está activa.</p></div>';
    } else {
        echo '<div class="notice notice-error"><p>❌ API de Funcionalidades no está disponible.</p></div>';
    }
}
```
