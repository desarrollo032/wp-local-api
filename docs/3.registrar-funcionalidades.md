# 3. Registrar Funcionalidades (`wp_register_feature`)

> **Guía completa para registrar funcionalidades usando la función `wp_register_feature()`.**

---

## Firma de la Función

```php
wp_register_feature( array|WP_Feature $feature_args ): bool
```

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `$feature_args` | `array` \| `WP_Feature` | Un array de argumentos que define la funcionalidad, o una instancia de la clase `WP_Feature`. |

**Retorno:** (`bool`) `true` si la funcionalidad fue registrada exitosamente, `false` en caso de fallo (por ejemplo, argumentos inválidos, ID duplicado).

---

## Parámetros Explicados

El array `$feature_args` acepta las siguientes claves:

### `id` (string, **Requerido**)

Identificador único para la funcionalidad.

- **Convención:** Usa un prefijo de namespace, como `mi-plugin/nombre-funcionalidad` o `woocommerce/obtener-producto`.
- **Formato:** Solo debe contener caracteres alfanuméricos en minúsculas (`a-z`, `0-9`), guiones (`-`) y barras diagonales (`/`) para namespacing. No puede comenzar o terminar con `/`, y no puede contener `//`.
- **Nota:** El prefijo de tipo (por ejemplo, `recurso-`, `herramienta-`) se añade _automáticamente_ al recuperar el ID completo de la funcionalidad vía `$feature->get_id()` o al encontrar funcionalidades. Se registra _sin_ el prefijo de tipo.

### `name` (string, **Requerido**)

Nombre legible para la funcionalidad. Usado para propósitos de visualización. Debe ser traducible.

### `description` (string, **Requerido**)

Descripción detallada de lo que hace la funcionalidad, su propósito y potencialmente sus parámetros o valores de retorno. Crucial para que los agentes de IA entiendan cómo y cuándo usar la funcionalidad. Debe ser traducible.

### `type` (string, **Requerido**)

El tipo de funcionalidad. Debe ser uno de:

| Tipo | Constante | Descripción |
|------|-----------|-------------|
| `recurso` | `WP_Feature::TYPE_RESOURCE` | Para recuperar datos (solo lectura). Equivale a `GET` en REST. |
| `herramienta` | `WP_Feature::TYPE_TOOL` | Para realizar acciones o modificar datos. Equivale a `POST`, `PUT`, `PATCH`, `DELETE` en REST. |

### `callback` (callable | null)

La función PHP o método a ejecutar cuando se llama a esta funcionalidad vía `$feature->call()`.

- Si es `null` o no se proporciona, llamar a la funcionalidad puede que no haga nada a menos que sea un `rest_alias`.
- El callback recibe un argumento: `$context` (un array asociativo de parámetros de entrada).
- El callback debe devolver el resultado de la operación de la funcionalidad o un objeto `WP_Error` en caso de fallo.

### `input_schema` (array)

Definición JSON Schema que describe los parámetros de entrada esperados (`$context`) para el `callback`. Usado para validación y documentación. Por defecto es un array vacío.

### `output_schema` (array)

Definición JSON Schema que describe el formato esperado de los datos devueltos por el `callback`. Usado para validación y documentación. Por defecto es un array vacío.

### `permission_callback` (callable | null)

Función callback para verificar si el usuario actual tiene permiso para ejecutar esta funcionalidad.

- Debe devolver `true` si el usuario tiene permiso, `false` o un objeto `WP_Error` de lo contrario.
- Por defecto es un callback que devuelve `false` si no se especifica. Usa `__return_true` para funcionalidades públicamente accesibles.
- Para funcionalidades `rest_alias`, esto a menudo se infiere del callback de permiso del endpoint REST aliasado.

### `is_eligible` (callable | null)

Función callback para determinar si la funcionalidad está disponible en el contexto actual.

- Debe devolver `true` si la funcionalidad es elegible, `false` de lo contrario.
- Útil para funcionalidades que dependen de otros plugins activos, roles de usuario específicos o configuraciones particulares del sitio.
- Por defecto es un callback que devuelve `true` (siempre elegible) si no se especifica.

### `categories` (string[])

Array de slugs de categorías para asociar con la funcionalidad. Ayuda a organizar y filtrar funcionalidades. Las categorías se crean implícitamente si no existen.

### `meta` (array)

Array asociativo para almacenar metadatos adicionales arbitrarios sobre la funcionalidad.

### `rest_alias` (string | false)

Si se establece como un string representando una ruta de API REST de WordPress, esta funcionalidad actuará como un alias para ese endpoint.

- Cuando se invoca `call()`, la API de Funcionalidades intentará mapear el contexto de entrada a los parámetros de la ruta REST y ejecutar la solicitud REST API correspondiente.
- El `input_schema`, `output_schema`, `permission_callback` y a veces el `callback` pueden inferirse automáticamente de la definición del endpoint REST. Puedes sobrescribirlos explícitamente.
- Por defecto es `false`.

## Ejemplos de Código

### Ejemplo 1: Registrar un Recurso Simple (Sin Callback)

```php
<?php
// Recupera información básica del sitio.
// Como no se proporciona callback, llamar a esta funcionalidad podría solo devolver el contexto de entrada
// o potencialmente nada útil a menos que se extienda mediante filtros.
// Un mejor enfoque para datos simples podría ser un callback o rest_alias.
wp_register_feature( array(
    'id'          => 'miplugin/info-basica',
    'name'        => __( 'Información Básica del Sitio', 'mi-plugin' ),
    'description' => __( 'Proporciona información muy básica del sitio.', 'mi-plugin' ),
    'type'        => WP_Feature::TYPE_RESOURCE,
    'categories'  => array( 'mi-plugin', 'info-sitio' ),
    'permission_callback' => '__return_true',
    'input_schema' => array(
        'type' => 'object',
    ),
    'output_schema' => array(),
) );
```

### Ejemplo 2: Registrar un Recurso Simple (Con Callback)

```php
<?php
function miplugin_obtener_nombre_tema_activo() {
    $tema = wp_get_theme();
    return $tema->get( 'Name' );
}

wp_register_feature( array(
    'id'          => 'miplugin/tema-activo',
    'name'        => __( 'Obtener Nombre del Tema Activo', 'mi-plugin' ),
    'description' => __( 'Recupera el nombre del tema actualmente activo.', 'mi-plugin' ),
    'type'        => WP_Feature::TYPE_RESOURCE,
    'callback'    => 'miplugin_obtener_nombre_tema_activo',
    'permission_callback' => '__return_true',
    'input_schema' => array(
        'type' => 'object'
    ),
    'output_schema' => array(
        'type' => 'string',
        'description' => __( 'El nombre del tema activo.', 'mi-plugin' ),
    ),
    'categories'  => array( 'mi-plugin', 'tema' ),
) );

// --- Uso ---
// $feature = wp_find_feature( 'recurso-miplugin/tema-activo' );
// $nombre_tema = $feature->call(); // Devuelve el string del nombre del tema
```

### Ejemplo 3: Registrar una Herramienta Simple (Con Callback y Esquema de Entrada)

```php
<?php
function miplugin_actualizar_opcion( $context ) {
    if ( ! isset( $context['nombre_opcion'] ) || ! isset( $context['valor_opcion'] ) ) {
        return new WP_Error( 'parametros_faltantes', 'Falta nombre_opcion o valor_opcion.' );
    }
    // Nota: Verificaciones de capacidad apropiadas deben estar en permission_callback
    return update_option( $context['nombre_opcion'], $context['valor_opcion'] );
}

wp_register_feature( array(
    'id'          => 'miplugin/actualizar-opcion',
    'name'        => __( 'Actualizar Opción de WordPress', 'mi-plugin' ),
    'description' => __( 'Actualiza el valor de una opción de WordPress en la base de datos.', 'mi-plugin' ),
    'type'        => WP_Feature::TYPE_TOOL,
    'callback'    => 'miplugin_actualizar_opcion',
    'permission_callback' => function() {
        return current_user_can( 'manage_options' );
    },
    'input_schema' => array(
        'type' => 'object',
        'properties' => array(
            'nombre_opcion' => array(
                'type' => 'string',
                'description' => __( 'El nombre de la opción a actualizar.', 'mi-plugin' ),
                'required' => true,
            ),
            'valor_opcion' => array(
                'type' => array( 'string', 'integer', 'boolean', 'array', 'object', 'null' ),
                'description' => __( 'El nuevo valor para la opción.', 'mi-plugin' ),
                'required' => true,
            ),
        ),
    ),
    'output_schema' => array(
        'type' => 'boolean',
        'description' => __( 'Verdadero si la opción fue actualizada, falso de lo contrario.', 'mi-plugin' ),
    ),
    'categories'  => array( 'mi-plugin', 'opciones' ),
) );

// --- Uso ---
// $feature = wp_find_feature( 'herramienta-miplugin/actualizar-opcion' );
// $resultado = $feature->call( [
//     'nombre_opcion' => 'blogdescription',
//     'valor_opcion' => '¡Una nueva descripción!'
// ] ); // Devuelve true o false
```

### Ejemplo 4: Registrar una Funcionalidad usando `rest_alias`

Este ejemplo (de `default-wp-features.php`) hace alias del endpoint REST core para obtener el usuario actual.

```php
<?php
wp_register_feature( array(
    'id'          => 'usuarios/yo', // ID base sin prefijo de tipo
    'name'        => __( 'Obtener usuario actual', 'wp-feature-api' ),
    'description' => __( 'Obtiene el usuario actual.', 'wp-feature-api' ),
    'rest_alias'  => '/wp/v2/users/me', // La ruta REST a la que hacer alias
    'categories'  => array( 'core', 'usuario', 'rest' ),
    'type'        => WP_Feature::TYPE_RESOURCE, // Esta es una solicitud GET
) );

// --- Uso ---
// $feature = wp_find_feature( 'recurso-usuarios/yo' );
// $datos_usuario = $feature->call(); // Ejecuta GET /wp/v2/users/me y devuelve objeto de usuario/error
```

### Ejemplo 5: Registrar una Funcionalidad con Comprobaciones de Elegibilidad

Este ejemplo muestra una funcionalidad solo disponible si WooCommerce está activo.

```php
<?php
wp_register_feature( array(
    'id'          => 'demo/informacion-woocommerce',
    'name'        => __( 'Información de WooCommerce', 'wp-feature-api-demo' ),
    'description' => __( 'Obtiene información básica sobre la configuración de WooCommerce.', 'wp-feature-api-demo' ),
    'type'        => WP_Feature::TYPE_RESOURCE,
    'categories'  => array( 'demo', 'woocommerce', 'informacion' ),
    'callback'    => function() {
        // Este código solo se ejecuta si is_eligible devuelve true
        if ( ! function_exists( 'WC' ) ) return null;
        return array(
            'version' => WC()->version,
            'moneda' => get_woocommerce_currency(),
        );
    },
    'is_eligible' => function () {
        // Solo elegible si existe la función WC (WooCommerce activo)
        return function_exists( 'WC' );
    },
    'permission_callback' => function() {
        // Ejemplo: Solo permitir administradores o gerentes de tienda
        return current_user_can( 'manage_woocommerce' );
    }
) );

// }
```

## Mejores Prácticas

| Práctica | Descripción |
|----------|-------------|
| **Namespacing** | Siempre usa un prefijo único relacionado con tu plugin o tema. |
| **Descripciones Claras** | Explica qué hace, qué espera y qué devuelve. |
| **Esquemas Completos** | Define `input_schema` y `output_schema` cuando sea posible. |
| **Permisos Apropiados** | No uses `__return_true` a menos que sea realmente público. |
| **Elegibilidad** | Usa `is_eligible` para funcionalidades condicionales. |
| **Categorización** | Agrupa funcionalidades lógicamente. |
