### 3. Registrar Features (`wp_register_feature`)

La forma principal de añadir funcionalidad a la Feature API es usando `wp_register_feature()`, normalmente enganchada al hook `wp_feature_api_init`.

**Firma de la función**

```php
wp_register_feature( array|WP_Feature $feature_args ): bool
```

- `$feature_args` (`array`|`WP_Feature`): Array de argumentos que definen la feature, o una instancia de `WP_Feature`.
- **Retorna:** (`bool`) `true` si la feature se registró correctamente, `false` en caso de fallo (p. ej., argumentos inválidos, ID duplicado).

**Parámetros explicados**

El array `$feature_args` acepta las siguientes claves (correspondientes a propiedades de `WP_Feature`):

- `id` (`string`, **Requerido**): Identificador único de la feature.
  - **Convención:** Usa un prefijo de namespace como `mi-plugin/feature-name` o `woocommerce/get-product`.
  - **Formato:** Solo letras minúsculas, dígitos (`0-9`), guiones (`-`) y barras (`/`). No puede empezar o terminar con `/` ni contener `//`.
  - **Nota:** El prefijo de `type` (p. ej. `resource-`, `tool-`) se añade _automáticamente_ cuando se obtiene el ID completo; regístralo sin ese prefijo.
- `name` (`string`, **Requerido**): Nombre legible de la feature. Debe ser traducible.
- `description` (`string`, **Requerido**): Descripción detallada del propósito, parámetros y resultados. Crucial para que agentes IA entiendan el uso.
- `type` (`string`, **Requerido**): Tipo de feature. Debe ser uno de:
  - `WP_Feature::TYPE_RESOURCE` ('resource'): Recuperación de datos (GET en REST).
  - `WP_Feature::TYPE_TOOL` ('tool'): Acciones o modificaciones (POST/PUT/PATCH/DELETE en REST).
- `callback` (`callable`|`null`): Función o método PHP a ejecutar al invocar la feature mediante `$feature->call()`.
  - Si es `null`, llamar la feature puede no hacer nada salvo que sea un `rest_alias`.
  - El callback recibe un argumento: `$context` (array asociativo de parámetros de entrada).
  - Debe retornar el resultado o un `WP_Error` en caso de fallo.
- `input_schema` (`array`): JSON Schema del input esperado (`$context`). Usado para validación y documentación. Por defecto vacío.
- `output_schema` (`array`): JSON Schema del output esperado. Por defecto vacío.
- `permission_callback` (`callable`|`null`): Callback que valida si el usuario actual puede ejecutar la feature.
  - Debe devolver `true` si tiene permisos, `false` o `WP_Error` en caso contrario.
  - Por defecto niega acceso si no se especifica; usar `__return_true` para features públicas (con precaución).
- `is_eligible` (`callable`|`null`): Callback que determina si la feature está disponible en el contexto actual.
  - Devuelve `true` si es elegible, `false` en caso contrario.
  - Útil cuando la feature depende de otros plugins, roles o ajustes.
  - Por defecto devuelve `true` si no está definido.
- `categories` (`string[]`): Array de slugs de categoría para organización y filtrado.
- `meta` (`array`): Array asociativo con metadatos arbitrarios.
- `rest_alias` (`string`|`false`): Si se establece con una ruta REST (p. ej. `/wp/v2/posts/(?P<id>[\d]+)`), la feature actúa como alias de ese endpoint.
  - Al invocar `call()`, la API intentará mapear el contexto a parámetros de la ruta REST y ejecutar la petición correspondiente.
  - `input_schema`, `output_schema` y `permission_callback` pueden inferirse del endpoint REST; puedes sobreescribirlos.
  - Por defecto `false`.

**Ejemplos de código**

**1. Registrar un Resource simple (sin callback)**

```php
wp_register_feature( array(
    'id'          => 'myplugin/basic-info',
    'name'        => __( 'Basic Site Info', 'my-plugin' ),
    'description' => __( 'Provides very basic site information.', 'my-plugin' ),
    'type'        => WP_Feature::TYPE_RESOURCE,
    'categories'  => array( 'my-plugin', 'site-info' ),
    'permission_callback' => '__return_true',
    'input_schema' => array(
        'type' => 'object',
    ),
    'output_schema' => array(),
) );
```

**2. Resource con callback**

```php
function myplugin_get_active_theme_name() {
    $theme = wp_get_theme();
    return $theme->get( 'Name' );
}

wp_register_feature( array(
    'id'          => 'myplugin/active-theme',
    'name'        => __( 'Get Active Theme Name', 'my-plugin' ),
    'description' => __( 'Retrieves the name of the currently active theme.', 'my-plugin' ),
    'type'        => WP_Feature::TYPE_RESOURCE,
    'callback'    => 'myplugin_get_active_theme_name',
    'permission_callback' => '__return_true',
    'input_schema' => array('type' => 'object'),
    'output_schema' => array(
        'type' => 'string',
        'description' => __( 'The name of the active theme.', 'my-plugin' ),
    ),
    'categories'  => array( 'my-plugin', 'theme' ),
) );

// Uso:
// $feature = wp_find_feature( 'resource-myplugin/active-theme' );
// $theme_name = $feature->call();
```

**3. Tool con callback y input_schema**

```php
function myplugin_update_option( $context ) {
    if ( ! isset( $context['option_name'] ) || ! isset( $context['option_value'] ) ) {
        return new WP_Error( 'missing_params', 'Missing option_name or option_value.' );
    }
    return update_option( $context['option_name'], $context['option_value'] );
}

wp_register_feature( array(
    'id'          => 'myplugin/update-option',
    'name'        => __( 'Update WordPress Option', 'my-plugin' ),
    'description' => __( 'Updates the value of a WordPress option in the database.', 'my-plugin' ),
    'type'        => WP_Feature::TYPE_TOOL,
    'callback'    => 'myplugin_update_option',
    'permission_callback' => function() {
        return current_user_can( 'manage_options' );
    },
    'input_schema' => array(
        'type' => 'object',
        'properties' => array(
            'option_name' => array(
                'type' => 'string',
                'description' => __( 'The name of the option to update.', 'my-plugin' ),
                'required' => true,
            ),
            'option_value' => array(
                'type' => array( 'string', 'integer', 'boolean', 'array', 'object', 'null' ),
                'description' => __( 'The new value for the option.', 'my-plugin' ),
                'required' => true,
            ),
        ),
    ),
    'output_schema' => array(
        'type' => 'boolean',
        'description' => __( 'True if the option was updated, false otherwise.', 'my-plugin' ),
    ),
    'categories'  => array( 'my-plugin', 'options' ),
) );

// Uso:
// $feature = wp_find_feature( 'tool-myplugin/update-option' );
// $result = $feature->call( [ 'option_name' => 'blogdescription', 'option_value' => 'A new tagline!' ] );
```

```php
wp_register_feature( array(
```

**5. Feature con checks de elegibilidad**

```php
wp_register_feature( array(
    'id'          => 'demo/woocommerce-info',
    'name'        => __( 'WooCommerce Information', 'wp-feature-api-demo' ),
    'description' => __( 'Get basic information about the configuration of WooCommerce.', 'wp-feature-api-demo' ),
    'type'        => WP_Feature::TYPE_RESOURCE,
    'categories'  => array( 'demo', 'woocommerce', 'information' ),
    'callback'    => function() {
        if ( ! function_exists( 'WC' ) ) return null; 
        return array(
            'version' => WC()->version,
            'currency' => get_woocommerce_currency(),
        );
    },
    'is_eligible' => function () {
        return function_exists( 'WC' );
    },
    'permission_callback' => function() {
        return current_user_can( 'manage_woocommerce' );
    }
) );
```
