# 5. Puntos Finales REST

> **Referencia completa de los endpoints REST API expuestos por la API de Funcionalidades.**

---

## Información General

La API de Funcionalidades expone endpoints REST para descubrir e interactuar con funcionalidades registradas remotamente.

| Propiedad | Valor |
|-----------|-------|
| **Namespace** | `wp/v2` |
| **Ruta Base** | `/features` |

---

## Autenticación

> ⚠️ **Nota:** Por defecto, estos endpoints requieren que un usuario esté conectado mediante autenticación estándar de cookies de WordPress. La autenticación puede extenderse o modificarse usando filtros estándar de autenticación de la REST API de WordPress si es necesario (por ejemplo, para Contraseñas de Aplicación o OAuth).

---

## Endpoints Disponibles

### 1. Listar Funcionalidades: `GET /wp/v2/features`

Recupera una lista paginada de todas las funcionalidades registradas y *elegibles*.

#### Parámetros de Consulta Soportados

| Parámetro | Tipo | Valor Predeterminado | Descripción |
|-----------|------|---------------------|-------------|
| `page` | integer | 1 | Número de página. |
| `per_page` | integer | 10 (máx: 100) | Elementos por página. |
| `fields` | string | todos | Lista de campos separados por comas a incluir (ej: `id,name,description`). |
| `type[]` | string | - | Filtrar por tipo (ej: `type[]=resource&type[]=tool`). |
| `categories[]` | string | - | Filtrar por slug de categoría (ej: `categories[]=core&categories[]=post`). |
| `location[]` | string | - | Filtrar por ubicación (ej: `location[]=server`). |
| `search` | string | - | Término de búsqueda. |

#### Encabezados de Respuesta

- `X-WP-Total`: Total de funcionalidades encontradas.
- `X-WP-TotalPages`: Total de páginas.
- `Link`: Encabezados para navegación `prev`/`next`.

#### Ejemplo de Solicitud

```bash
GET /wp-json/wp/v2/features?per_page=5&type[]=tool&categories[]=woocommerce
```

---

### 2. Listar Categorías: `GET /wp/v2/features/categories`

Recupera una lista de todas las categorías de funcionalidades registradas.

#### Ejemplo de Solicitud

```bash
GET /wp-json/wp/v2/features/categories
```

---

### 3. Obtener Categoría Específica: `GET /wp/v2/features/categories/{id}`

Recupera detalles de una sola categoría por su slug.

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `{id}` | string | El slug de la categoría (ej: `core`, `woocommerce`). |

#### Ejemplo de Solicitud

```bash
GET /wp-json/wp/v2/features/categories/woocommerce
```

---

### 4. Obtener Funcionalidad Específica: `GET /wp/v2/features/{feature-id}`

Recupera detalles de una sola funcionalidad por su ID completo (incluyendo prefijo de tipo).

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `{feature-id}` | string | El ID completo de la funcionalidad (ej: `recurso-usuarios/yo`). |

#### Ejemplo de Solicitud

```bash
GET /wp-json/wp/v2/features/recurso-usuarios/yo
```

---

### 5. Ejecutar Funcionalidad: `POST|GET /wp/v2/features/{feature-id}/run`

Ejecuta la funcionalidad especificada.

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `{feature-id}` | string | El ID completo de la funcionalidad (ej: `herramienta-miplugin/actualizar-opcion`). |

#### Método HTTP

| Tipo de Funcionalidad | Método HTTP |
|-----------------------|-------------|
| `herramienta` | `POST` |
| `recurso` | `GET` |

#### Cuerpo de Solicitud (JSON para POST)

```json
{
    "option_name": "blogdescription",
    "option_value": "Actualizado vía REST"
}
```

#### Parámetros de Consulta (para recursos GET)

Añade parámetros a la URL si la funcionalidad los espera (a menudo derivados de rutas `rest_alias`).

```bash
GET /wp-json/wp/v2/features/recurso-post/run?id=1
```

> 💡 **Nota:** Los REST Alias mapean parámetros de URL automáticamente.

#### Ejemplos Completos

**Ejemplo de Herramienta:**
```bash
POST /wp-json/wp/v2/features/herramienta-miplugin/actualizar-opcion/run
Content-Type: application/json

{
    "option_name": "blogdescription",
    "option_value": "¡Nueva descripción del blog!"
}
```

**Ejemplo de Recurso:**
```bash
GET /wp-json/wp/v2/features/recurso-miplugin/tema-activo/run
```

## Ejemplo de Respuesta

### Listar Funcionalidades

```json
{
    "code": "rest_ok",
    "data": [
        {
            "id": "recurso-usuarios/yo",
            "name": "Obtener usuario actual",
            "description": "Obtiene el usuario actual autenticado.",
            "type": "resource",
            "categories": ["core", "user", "rest"],
            "input_schema": {
                "type": "object",
                "properties": {}
            },
            "output_schema": {
                "type": "object",
                "properties": {
                    "id": { "type": "integer" },
                    "username": { "type": "string" },
                    "email": { "type": "string" }
                }
            }
        }
    ],
    "total": 1,
    "total_pages": 1
}
```

### Ejecutar Herramienta (Éxito)

```json
{
    "success": true,
    "message": "Opción actualizada correctamente"
}
```

### Ejecutar Herramienta (Error)

```json
{
    "code": "permission_denied",
    "message": "No tienes permisos para realizar esta acción.",
    "data": {
        "status": 403
    }
}
```

---

## Códigos de Estado HTTP

| Código | Significado |
|--------|-------------|
| `200` | Éxito |
| `400` | Solicitud incorrecta (parámetros inválidos) |
| `401` | No autenticado |
| `403` | Permiso denegado |
| `404` | Funcionalidad no encontrada |
| `500` | Error del servidor |
