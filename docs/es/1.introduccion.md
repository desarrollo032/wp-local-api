# 1. Introducción y Descripción General

> **Información general sobre la API de Funcionalidades de WordPress y sus capacidades.**

---

## ¿Qué es la API de Funcionalidades?

La API de Funcionalidades de WordPress proporciona una forma estandarizada de registrar y descubrir unidades distintas de funcionalidad dentro de un sitio WordPress. Estas unidades, llamadas **"Funcionalidades" (Features)**, pueden representar acciones de recuperación de datos (**Recursos**) o acciones de modificación/creación de datos (**Herramientas**).

Acts como un registro central, facilitando que diferentes partes de WordPress, plugins de terceros, temas y sistemas externos (como agentes de IA) comprendan e interactúen con las capacidades disponibles en un sitio específico.

---

## Conceptos Clave

| Concepto | Descripción |
|----------|-------------|
| **Funcionalidad (Feature)** | Una pieza distintiva de funcionalidad con ID único, nombre legible, descripción, definiciones de entrada/salida (usando JSON Schema), permisos y código asociado. |
| **Recurso (Resource)** | Tipo de Funcionalidad para *recuperar* datos (operaciones de solo lectura). Típicamente corresponde a solicitudes `GET` en REST. |
| **Herramienta (Tool)** | Tipo de Funcionalidad para *realizar acciones* o *modificar* datos (operaciones de escritura/creación/actualización/eliminación). Típicamente corresponde a solicitudes `POST`, `PUT`, `PATCH`, `DELETE` en REST. |
| **Registro (Registry)** | Objeto central singleton (`WP_Feature_Registry`) que contiene todas las funcionalidades registradas. Proporciona métodos para registrar, eliminar, encontrar y consultar funcionalidades. |
| **Repositorio** | Mecanismo de almacenamiento subyacente para el registro (por ejemplo, en memoria, base de datos). La API define una interfaz (`WP_Feature_Repository_Interface`) permitiendo diferentes backends de almacenamiento. El valor predeterminado es en memoria (`WP_Feature_Repository_Memory`). |
| **Callback** | La función PHP o método que se ejecuta cuando se llama a una Funcionalidad. |
| **Esquema (Schema)** | Definiciones JSON Schema para la entrada esperada (`input_schema`) y la salida devuelta (`output_schema`) de una Funcionalidad. Permite validación y ayuda a los agentes a entender cómo usar la funcionalidad. |
| **Categoría** | Una forma de agrupar funcionalidades relacionadas para mejor organización y descubrimiento. |
| **Elegibilidad (`is_eligible`)** | Función callback que determina si una funcionalidad debe estar disponible en el contexto actual. |

---

## Objetivos y Beneficios

- **Estandarización:** Proporciona una forma única y consistente de exponer las capacidades del sitio.
- **Descubribilidad:** Permite que sistemas (como agentes de IA o componentes de UI) encuentren fácilmente acciones y fuentes de datos disponibles.
- **Conciencia del Contexto:** Permite que las funcionalidades estén disponibles condicionalmente según el entorno o contexto del usuario (`is_eligible`).
- **Reutilización:** Las funcionalidades definidas una vez pueden ser usadas por múltiples consumidores (por ejemplo, una llamada PHP directa, una solicitud REST API, un agente de IA).
- **Extensibilidad:** Diseñado para que plugins de terceros y temas registren sus propias funcionalidades junto con las del core.
- **Fundamento para IA:** Proporciona los bloques de construcción necesarios (herramientas y recursos) para agentes de IA que operan dentro de WordPress o interactúan con él.

---

## Relación con MCP (Model Context Protocol)

> ⚠️ **Nota Importante:** La API de Funcionalidades **NO ES** [MCP](https://modelcontextprotocol.io/). MCP es una especificación de protocolo para cómo los agentes de IA y las capacidades interactúan *entre sistemas*.

La API de Funcionalidades es un sistema *nativo de WordPress* para *registrar y gestionar capacidades dentro de un solo sitio WordPress*. Está diseñada para ser **compatible** con MCP. El patrón previsto es:

1. Plugins, temas y core registran su funcionalidad como Funcionalidades usando esta API.
2. Un **Adaptador MCP** separado (otro plugin o parte de un sistema de IA más grande) lee las funcionalidades del registro de la API de Funcionalidades. Aquí está el [Adaptador MCP de Automattic](https://github.com/Automattic/wordpress-mcp), que soporta la API de Funcionalidades.
3. El Adaptador MCP expone estas Funcionalidades registradas como Herramientas y Recursos de MCP a través del protocolo MCP.

Esta separación permite que la API de Funcionalidades permanezca enfocada en la integración de WordPress mientras permite la interoperabilidad con el ecosistema MCP más amplio a través de una capa adaptadora.
