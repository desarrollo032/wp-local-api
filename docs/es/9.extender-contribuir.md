# 9. Extender y Contribuir

> **Guía para desarrolladores de terceros que desean registrar funcionalidades en sus plugins y temas.**

---

## Registrar Funcionalidades en Plugins/Temas

Se anima a los desarrolladores de terceros a registrar funcionalidades proporcionadas por sus plugins o temas.

### Pasos Recomendados

| Paso | Acción | Descripción |
|------|--------|-------------|
| 1 | **Hook** | Usa el hook de acción `init`. |
| 2 | **Función** | Llama a `wp_register_feature()` con un array definiendo tu funcionalidad. |
| 3 | **Namespacing** | **Crucialmente**, añade un prefijo a tu ID de funcionalidad con un namespace único específico de tu plugin/tema. |
| 4 | **Descripciones** | Escribe descripciones claras y detalladas explicando qué hace la funcionalidad, sus parámetros y qué devuelve. |
| 5 | **Esquemas** | Define `input_schema` y `output_schema` usando JSON Schema cuando sea aplicable. |
| 6 | **Permisos** | Implementa un `permission_callback` para asegurar que solo usuarios autorizados puedan ejecutar la funcionalidad. |
| 7 | **Elegibilidad** | Usa `is_eligible` si la funcionalidad depende de condiciones específicas. |
| 8 | **Categorías** | Asigna categorías relevantes, potencialmente incluyendo el namespace de tu plugin como categoría. |

## Ejemplo Completo

```php
<?php
/**
 * Plugin Name: Mi Plugin SEO
 * Descripción: Registra funcionalidades SEO con la API de Funcionalidades.
 */

// Exit if accessed directly
if ( ! defined( 'ABSPATH' ) ) {
    exit;
}

/**
 * Registra las funcionalidades del plugin SEO.
 */
function mi_plugin_seo_registrar_funcionalidades() {
    // Verificar que la API de Funcionalidades esté disponible
    if ( ! function_exists( 'wp_register_feature' ) || ! class_exists( '\WP_Feature' ) ) {
        return;
    }

    wp_register_feature( array(
        'id'          => 'mi-seo-plugin/obtener-meta-descripcion',
        'name'        => __( 'Obtener Descripción Meta SEO', 'mi-seo-plugin' ),
        'description' => __( 'Recupera la descripción meta SEO para un ID de publicación dado.', 'mi-seo-plugin' ),
        'type'        => WP_Feature::TYPE_RESOURCE,
        'callback'    => 'mi_plugin_seo_callback_obtener_meta_descripcion',
        'permission_callback' => '__return_true', // O verificación de capacidad específica
        'input_schema' => array(
            'type' => 'object',
            'properties' => array(
                'post_id' => array(
                    'type' => 'integer',
                    'description' => 'ID de la publicación.'
                )
            ),
            'required' => ['post_id']
        ),
        'output_schema' => array(
            'type' => 'string',
            'description' => 'El texto de la descripción meta.'
        ),
        'categories'  => array( 'mi-seo-plugin', 'seo', 'post' ),
    ) );
}
add_action( 'init', 'mi_plugin_seo_registrar_funcionalidades' );

/**
 * Callback para obtener la descripción meta SEO.
 *
 * @param array $context Contexto con post_id.
 * @return string|WP_Error La descripción meta o error.
 */
function mi_plugin_seo_callback_obtener_meta_descripcion( $context ) {
    $post_id = $context['post_id'] ?? null;
    
    if ( ! $post_id ) {
        return new WP_Error( 'post_id_faltante', 'Se requiere el ID de la publicación.' );
    }
    
    return get_post_meta( $post_id, '_mi_seo_meta_description', true );
}
```

## Mejores Prácticas

### Namespacing

> ⚠️ **Importante:** Usa prefijos de namespace únicos para evitar colisiones.

| Bueno | Malo |
|-------|------|
| `mi-plugin/obtener-datos` | `obtener-datos` |
| `woocommerce/obtener-producto` | `producto` |
| `mitema/obtener-colores-personalizados` | `colores` |

### Descripciones

Las descripciones deben ser:

- **Claras:** Explicar exactamente qué hace la funcionalidad.
- **Completas:** Incluir información sobre parámetros y valores de retorno.
- **Orientadas a IA:** Ser comprensibles para agentes de IA.

**Ejemplo de Buena Descripción:**
```php
'description' => __( 'Actualiza el título SEO de una publicación. Requiere el ID de la publicación y el nuevo título. Devuelve true en éxito o WP_Error si la publicación no existe.', 'mi-plugin' ),
```

### Esquemas

| Aspecto | Recomendación |
|---------|---------------|
| **input_schema** | Definir siempre que la funcionalidad acepte parámetros. |
| **output_schema** | Definir para que los consumidores sepan qué esperar. |
| **required** | Marcar campos obligatorios en el array `required`. |
| **descriptions** | Incluir descripciones para cada propiedad. |

### Permisos

| Situación | Callback |
|-----------|----------|
| Pública | `__return_true` |
| Administrador | `function() { return current_user_can( 'manage_options' ); }` |
| Editor | `function() { return current_user_can( 'edit_posts' ); }` |
| Personalizado | Tu propia lógica de capacidad |

### Elegibilidad

```php
'is_eligible' => function () {
    // Solo disponible si WooCommerce está activo
    return function_exists( 'WC' );
},
```

```php
'is_eligible' => function () {
    // Solo disponible si una configuración está habilitada
    return get_option( 'mi_plugin_enabled', false ) === true;
},
```

## Contribuir al Proyecto Principal

Si la API de Funcionalidades evoluciona hacia una característica central o un plugin canónico, las guías de contribución se establecerían a través del proceso de contribución estándar de WordPress.

### Canales de Contribución

| Canal | Uso |
|-------|-----|
| **GitHub** | Reportes de bugs, pull requests, discusiones. |
| **Trac** | Contribuciones al core de WordPress (si se integra). |
| **Foros** | Soporte y preguntas de la comunidad. |

### Directrices Generales

1. **Sigue los estándares de codificación** de WordPress.
2. **Escribe pruebas** para nuevas funcionalidades.
3. **Documenta** tu código y cambios.
4. **Usa mensajes de commit** descriptivos.
5. **Haz PRs pequeñas y enfocadas**.

## Verificación de Dependencias

Antes de registrar funcionalidades, siempre verifica que la API de Funcionalidades esté disponible:

```php
<?php
function mi_plugin_registrar_funcionalidades() {
    // Verificar que las funciones existen
    if ( ! function_exists( 'wp_register_feature' ) ) {
        add_action( 'admin_notices', function() {
            echo '<div class="notice notice-error"><p>';
            echo esc_html__( 'Mi Plugin requiere la API de Funcionalidades de WordPress.', 'mi-plugin' );
            echo '</p></div>';
        } );
        return;
    }
    
    // Proceder con el registro...
    wp_register_feature( array( /* ... */ ) );
}
add_action( 'init', 'mi_plugin_registrar_funcionalidades', 20 );
```
