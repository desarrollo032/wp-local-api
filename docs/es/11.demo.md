# 11. Demostración del Agente de la API de Funcionalidades

> **Explora el plugin de demostración que muestra cómo los agentes de IA pueden interactuar con WordPress usando la API de Funcionalidades.**

---

## Descripción General

La Demostración del Agente de la API de Funcionalidades de WordPress (`demo/wp-feature-api-agent/`) es un plugin de ejemplo funcional que muestra cómo la API de Funcionalidades de WordPress puede permitir que los agentes de IA interactúen con WordPress.

Demuestra una arquitectura donde un orquestador de agentes del lado del cliente se comunica con un proxy PHP del backend, que a su vez interactúa con ChatGPT. El concepto clave ilustrado es cómo las Funcionalidades de WordPress (tanto del lado del servidor como del cliente) pueden exponerse como "herramientas" para que el LLM las use, permitiendo control conversacional sobre la funcionalidad de WordPress.

---

## Conceptos Clave Ilustrados

La demostración ilustra varios conceptos centrales de la API de Funcionalidades:

| Concepto | Descripción |
|----------|-------------|
| **Registro de Funcionalidades** | Ejemplos de registro de funcionalidades de tipo recurso y herramienta. |
| **Comunicación Servidor-Cliente** | Demostrando cómo el servidor y el cliente pueden intercambiar capacidades. |
| **Descubrimiento de Funcionalidades** | Cómo las funcionalidades pueden descubrirse y ejecutarse programáticamente. |
| **Definición de Esquemas** | Usar JSON Schema para definir entradas y salidas. |
| **Control de Permisos** | Implementar callbacks de permisos para funcionalidades. |
| **Comprobaciones de Elegibilidad** | Usar verificaciones condicionales para determinar si una funcionalidad debe estar disponible. |
| **Interfaz Conversacional** | Una implementación práctica mostrando cómo los agentes de IA pueden interactuar con funcionalidades. |

### Componentes del Plugin

| Componente | Archivo | Propósito |
|------------|---------|-----------|
| **Plugin Core** | `wp-feature-api-agent.php` | Inicializa la demostración, carga dependencias y encola assets. |
| **Proxy Backend** | `includes/class-wp-ai-api-proxy.php` | Maneja solicitudes REST API del cliente, se comunica con la API de LLM configurada e inyecta herramientas descubiertas. |
| **Configuración** | `includes/class-wp-ai-api-options.php` | Gestiona configuraciones para claves de API. |
| **Registro de Funcionalidades** | `includes/class-wp-feature-register.php` | Registra funcionalidades específicas del servidor de la demostración (ej: `demo/info-sitio`). |
| **Orquestador del Cliente** | `src/agent/orchestrator.ts` | Gestiona el bucle de conversación, llama al proxy del backend, procesa respuestas del LLM (incluyendo llamadas a herramientas) y coordina la ejecución de herramientas. |
| **Ejecutor de Herramientas** | `src/agent/tool-executor.ts` | Responsible de ejecutar herramientas específicas (funcionalidades) cuando las solicita el orquestador. |
| **Proveedor de Herramientas** | `src/agent/wp-feature-tool-provider.ts` | Descubre las funcionalidades de WordPress disponibles (servidor y cliente) y las pone a disposición del `ToolExecutor`. |
| **Interfaz UI React** | `src/index.tsx`, `src/components/`, `src/context/` | Proporciona la interfaz conversacional (ventana de chat) en el admin de WordPress, impulsada por componentes React y contexto para gestión de estado. |

## Funcionalidades del Lado del Servidor

La demostración registra una sencilla funcionalidad de recurso del lado del servidor:

```php
<?php
// En includes/class-wp-feature-register.php
wp_register_feature(
    array(
        'id'          => 'demo/info-sitio',
        'name'        => __( 'Información del Sitio', 'wp-feature-api-agent' ),
        'description' => __( 'Obtiene información básica sobre el sitio WordPress. Esto incluye el nombre, descripción, URL, versión, idioma, zona horaria, formato de fecha, formato de hora, plugins activos y tema activo.', 'wp-feature-api-agent' ),
        'type'        => WP_Feature::TYPE_RESOURCE,
        'categories'  => array( 'demo', 'sitio', 'informacion' ),
        'callback'    => array( $this, 'site_info_callback' ),
    )
);
```

Mientras que esta demostración solo incluye un ejemplo específico, el `WpFeatureToolProvider` descubrirá automáticamente *cualquier* funcionalidad registrada vía `wp_register_feature` o los mecanismos de registro de funcionalidades del lado del cliente, haciéndolas disponibles como herramientas para el agente.

## Implementación del Lado del Cliente

El lado del cliente está construido con React y TypeScript:

### Componentes de UI

| Componente | Propósito |
|------------|-----------|
| `ChatApp.tsx` | Proporciona la estructura principal de la ventana de chat. |
| `ChatMessage.tsx` | Renderiza mensajes individuales (usuario, asistente, herramientas). |

### Gestión de Estado

| Componente | Propósito |
|------------|-----------|
| `ConversationProvider.tsx` | Usa React Context para gestionar el historial de mensajes, estado de carga y proporciona la función `sendMessage`. También inicializa el orquestador del agente y el ejecutor de herramientas. |

### Lógica del Agente

| Componente | Propósito |
|------------|-----------|
| `orchestrator.ts` | Contiene la función generadora asíncrona `processQuery` que maneja el bucle de conversación, llamadas a API y procesamiento de llamadas a herramientas. |
| `tool-executor.ts` | Define la clase `ToolExecutor` responsable de llamar al proveedor de herramientas apropiado. |
| `wp-feature-tool-provider.ts` | Implementa la lógica para encontrar y ejecutar funcionalidades de WordPress basándose en su ID, interactuando con la API de Funcionalidades central. |

La demostración incluye una interfaz conversacional implementada como una aplicación React embebida en el área de administración de WordPress. Esta interfaz:

Esta interfaz demuestra cómo los agentes de IA pueden aprovechar las funcionalidades de WordPress descubiertas a través de la API para cumplir solicitudes del usuario.

## Ideas y Patrones Clave

### Uso de Herramientas LLM vía API de Funcionalidades

La idea central es usar la API de Funcionalidades como un puente para permitir que los LLMs interactúen con la funcionalidad de WordPress de una manera estructurada. Las funcionalidades se convierten en herramientas que el LLM puede solicitar, permitiendo acciones complejas basadas en lenguaje natural.

### Orquestación de Agentes

La demostración implementa un patrón de orquestación del lado del cliente. El orquestador gestiona el flujo entre la interfaz de usuario, el proxy del backend LLM y el sistema de ejecución de herramientas. Esto mantiene el cliente responsivo y permite interacciones de múltiples pasos complejas que involucran llamadas a herramientas.

### Patrón de Proxy Backend

Usar un proxy PHP del backend (`WP_AI_API_Proxy`) proporciona varios beneficios:

| Beneficio | Descripción |
|-----------|-------------|
| **Seguridad** | Las claves de API para el servicio LLM externo se mantienen en el servidor, no expuestas al navegador. |
| **Control** | El proxy puede aplicar reglas, inyectar contexto (como herramientas disponibles) y gestionar interacciones de API. |
| **Abstracción** | El código del lado del cliente interactúa con un endpoint consistente de la REST API de WordPress, independientemente del servicio LLM específico usado en el backend. |

## Instalación y Uso

### Requisitos Previos

1. Plugin principal **WordPress Feature API** instalado y activado.
2. Una clave de API de OpenAI (u otro proveedor de LLM compatible).

### Pasos de Instalación

1. Asegúrate de que el plugin principal de WordPress Feature API esté instalado y activado.
2. **Configura el Servicio de IA:** Navega a la página de configuración ajoutée por la demostración (bajo "Ajustes" > "Feature API Demo") e introduce tu clave de API para OpenAI.
3. Activa el plugin de demostración.

### Ejemplos de Uso

Una vez configurado, intenta solicitudes en lenguaje natural como:

| Solicitud | Funcionalidad Usada |
|-----------|---------------------|
| "Dime sobre este sitio." | `demo/info-sitio` |
| "¿Qué versión de WordPress es esta?" | `demo/info-sitio` |
| "Crea una publicación basada en mi información del sitio." | Funcionalidades combinadas |
